# Preguntas
 - ¿Qué fue lo que más te gustó de trabajar con GitHub Desktop?

 - ¿Qué te pareció más difícil?
    la parte de github desktop porque aveces no se clonaban los repositorios 
 - ¿Para qué crees que sirve un archivo .gitignore?
    para ignorar otros archivos y que no se suban a la web de github 
 - ¿Por qué es importante escribir buenos mensajes en los commits?
    para la buena organizacion de los comentarios, para facilitar y evidenciar los cambios hechos  
## Ejemplo de cómo agregarlo:
  Se llenan los campos de summary y descripcion (opcional)
  Summary: Actualización del README con información del proyecto.
  Descripcion: se especifican los cambios hechos
  Se hace clic en "Commit to main".
  Después clic en "Push origin".
  Y asi finalmente se sube el commit.
